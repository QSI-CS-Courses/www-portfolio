What is Command Line Interface (CLI)?

The Command Line Interface (CLI) is a text-based interface for interacting with your computer's operating system. Instead of clicking icons and using a graphical interface, you type commands to perform tasks.

Text-Based

Interact with your computer through typed commands rather than graphical elements.

Direct Control

Execute commands directly on the operating system without intermediate layers.

Developer Essential

A fundamental tool that every developer needs in their toolkit.

Why Use the Command Line?

Efficiency

Precision

Automation

Remote Access

What is HTML?

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It uses "tags" to structure content on the web.

Document Structure Tags

html The Root Element

The root element of an HTML page. All other elements are contained within this tag.

head Document Metadata

Contains meta information about the document. Not displayed on the page. Includes title, styles, scripts, and metadata.

title Page Title

Sets the title of the webpage (shown in the browser tab).

style CSS Rules

Contains CSS rules for styling the page. Defines how elements should look.

body Visible Content

Contains all the content that is visible on the webpage. Images, text, links, lists, etc. go here.

$ echo "HTML provides the structure of web pages"
HTML provides the structure of web pages

Structural Elements

header Page Header

Represents introductory content or a set of navigational links. Usually contains the site title, logo, or navigation. Found at the top of the page with the "Saitama" heading.

main Main Content

Represents the dominant content of the body. There should be only one main element per page. Contains the image carousel and information about Saitama.

footer Page Footer

Represents a footer for a document or section. Typically contains author information, copyright, or contact info. Found at the bottom with "Made with ‚ù§Ô∏è in WWW Bootcamp".

div Generic Container

A generic container for flow content. Has no effect on content or layout until styled with CSS. Used to group elements for styling purposes. Look for the "hero-card" div that contains the image and info sections.

Semantic HTML

Using appropriate structural elements like header, main, and footer makes your code more meaningful and accessible.

Content Elements

h1 through h6 Headings

Headings, with h1 being the most important. Used to structure content hierarchically. Should be used in order (don't skip from h1 to h3). Examples: "Saitama" (h1), "About Me" (h2), "My Powers" (h3).

p Paragraph

Represents a paragraph of text. Block-level element with space above and below. Contains the description of Saitama's training routine.

img Image

Embeds an image in the document. Must have src (source) and alt (alternative text) attributes. Self-closing tag. The Saitama images in the carousel.

ul and li Lists

ul: Unordered list (bullet points). li: List item. Used for grouping related items. The list of Saitama's powers.

button Button

Represents a clickable button. Can be used in forms or anywhere a clickable action is needed. Can contain text or images. The "Contact Me" button.

CSS Exploration - Saitama Portfolio

What is CSS?

CSS (Cascading Style Sheets) is the language used to describe the presentation of web pages. It controls how HTML elements look - their colors, fonts, spacing, layout, and animations.

CSS in the Saitama Portfolio

In the Saitama portfolio, CSS is contained within the <style> tags in the <head> section of the HTML document. This is called "internal CSS".

CSS Structure

CSS consists of rules that have this format:

selector {
  • property: value;
  • property: value;
}

Example from Saitama's portfolio:

header {
  • text-align:center;
  • padding: 2rem 1rem;
  • background: linear-gradient(135deg, #ff9a9e 0%, #fad0c4 100%);
  • color: white;
}

CSS Selectors

Selectors are patterns used to select HTML elements you want to style:

Element Selectors

  • header - Selects all <header> elements
  • body - Selects <body> element
  • h1 - Selects all <h1> elements
  • button - Selects all <button> elements

Class Selectors

  • .hero-card - Selects elements with class="hero-card"
  • .info - Selects elements with class="info"

ID Selectors

  • #nav-btn - Selects element with id="contact-btn"
  • #contact-modal - Selects element with id="contact-modal"

Pseudo-selectors

  • button:hover - Styles applied when hovering over buttons
  • button.active - Styles for buttons with the "active" class

CSS Properties in Saitama Portfolio

Colors and Backgrounds

  • color - Text color (e.g., color: white;
  • background-color - Background color of an element
  • background - Complex backgrounds (like gradients)
  • --accent: #ff4757 - CSS variable for consistent color usage

Typography

  • font-family - Font type (e.g., 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif)
  • font-size - Size of text (e.g., 2.8rem, 1.2rem)
  • text-shadow - Shadow effect for text
  • line-height - Space between lines of text

Box Model Properties

  • margin - Space outside an element
  • padding - Space inside an element
  • border - Border around an element
  • border-radius - Rounded corners (e.g., border-radius: 16px;

Layout

  • display - How an element is displayed (flex, block, none, etc.)
  • position - Positioning method (relative, absolute, fixed)
  • width and height - Dimensions of elements
  • max-width - Maximum width an element can have

Flexbox (Modern Layout)

  • display: flex - Makes container a flex container
  • flex-direction - Direction of flex items (row, column)
  • justify-content - Alignment along main axis
  • align-items - Alignment along cross axis
  • gap - Space between flex items

CSS Specificity and Cascade

CSS follows the "Cascading" principle:

  1. Inline styles (in HTML) have highest priority
  2. ID selectors (#id)
  3. Class selectors (.class)
  4. Element selectors (div, p, h1)

When multiple rules apply to the same element, the one with higher specificity wins.
